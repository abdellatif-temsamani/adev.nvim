		     A D E V . N V I M    R E F E R E N C E    M A N U A L

Adev.nvim is a comprehensive Neovim distribution designed for developers who want
a complete, modern development environment out of the box. This manual provides
detailed information about installation, configuration, features, and usage.

==============================================================================
INTRODUCTION						*adev-introduction*

Adev.nvim provides:
- Modern plugin ecosystem with lazy loading
- Language Server Protocol (LSP) support for 30+ languages
- Advanced code completion and snippets
- Git integration and version control tools
- Customizable UI with themes and status lines
- Feature flags for experimental functionality
- Modular architecture for easy extension

==============================================================================
INSTALLATION AND SETUP					*adev-installation*

REQUIREMENTS						*adev-requirements*

- Neovim 0.11.0 or higher
- Git
- Node.js 20+ (for LSP servers and tools)
- Modern terminal with true color support
- Linux/macOS (Windows support experimental)

QUICK START						*adev-quick-start*

1. Clone the repository: >
	git clone https://github.com/abdellatif-temsamani/adev.nvim ~/.config/nvim
<
2. Start Neovim: >
	nvim
<
3. Install plugins: >
	:Lazy sync
<
4. Restart Neovim to complete setup

FIRST RUN						*adev-first-run*

On first startup, Adev.nvim will:
- Install Lazy.nvim plugin manager
- Download and configure all plugins
- Set up LSP servers automatically
- Run onboarding wizard
- Configure default settings

CONFIGURATION REFERENCE					*adev-configuration*

ADCONFIG SYSTEM						*adev-adconfig*

Adev.nvim uses a modular configuration system called ADConfig. The main
components are:

- lua/adev/init.lua          - Main setup function
- lua/adev/defaults.lua      - Default configuration options
- lua/adev/init-opts.lua     - User overrides (optional)
- lua/adev/plugins/          - Plugin specifications
- lua/adev/custom-plugins/   - User-added plugins

CUSTOMIZATION						*adev-customization*

To customize Adev.nvim, use the :ADConfig command to open the configuration file: >

:ADConfig
<

This will open lua/adev/init-opts.lua for editing. Do not modify init.lua or
lua/adev/init-opts.lua directly, as init.lua is auto-generated and changes may
be overwritten. Use :ADConfig instead.

Example configuration: >

return {
  git = {
    path = "/usr/bin/git"  -- Custom git path
  },
  ui = {
    borders = "rounded"    -- Border style
  },
  lsp = {
    enable = true          -- Enable LSP
  }
}
<

FEATURE FLAGS						*adev-feature-flags*

Control experimental features via flags using :ADConfig: >

:ADConfig
<

Then add to your configuration: >

return {
  flags = {
    experimental_adev_files = true,  -- Enable file management
    ai_assistant = {
      enabled = true
    }
  }
}
<

PLUGIN DOCUMENTATION					*adev-plugins*

PACKAGE MANAGER						*adev-package-manager*

Adev.nvim uses lazy.nvim as its plugin manager. lazy.nvim provides:

- Declarative plugin specifications
- Lazy loading for performance
- Plugin update management
- Dependency resolution
- UI for plugin management

The lazy.nvim configuration is defined in lua/adev/lazy.lua, which:
- Automatically installs lazy.nvim if missing
- Imports plugin specs from lua/adev/plugins/ and lua/adev/custom-plugins/
- Enables rocks/pkg support for Lua packages
- Configures caching and performance optimizations

ADEV-FILES						*adev-files*

Adev-files is an in-house experimental file and directory management system
designed to provide advanced file operations within Neovim. It serves as an
alternative to external file explorers with deeper Neovim integration.

FEATURES						*adev-files-features*

Current MVP features:
- Create new files and directories
- List files and directories with filtering

Planned features:
- Respect git ignored files
- Integration with LSP rename operations
- Move/copy/delete files and directories
- Multi-file selection and operations

ENABLING ADEV-FILES					*adev-files-enable*

To enable adev-files, use :ADConfig to edit your configuration and add: >

return {
  flags = {
    experimental_adev_files = true,
  }
}
<

When enabled, adev-files replaces the Snacks file explorer (<leader>n) and
provides its own file management interface.

KEYMAPS							*adev-files-keymaps*

| Mapping    | Description              |
|------------|--------------------------|
| <leader>na | Create new file/directory |
| <leader>nl | List files in directory   |

USAGE							*adev-files-usage*

Creating files: >
:lua require('adev-files').create()
<

Listing files: >
:lua require('adev-files').list()
<

The system is designed to work seamlessly with Neovim's buffer management and
provides a more integrated experience than traditional file explorers.

TROUBLESHOOTING AND FAQ				*adev-troubleshooting*

COMMON ISSUES						*adev-common-issues*

PLUGINS NOT LOADING					*adev-plugins-not-loading*

Problem: Plugins fail to install or load.
Solution:
1. Check internet connection
2. Run :Lazy sync
3. Check :Lazy log for errors
4. Ensure git is available

LSP SERVERS NOT WORKING				*adev-lsp-not-working*

Problem: Language servers don't start.
Solution:
1. Install Node.js 16+
2. Run :Mason to check server status
3. Check :LspInfo for diagnostics
4. Restart Neovim

SLOW STARTUP						*adev-slow-startup*

Problem: Neovim takes too long to start.
Solution:
1. Check :Lazy profile for slow plugins
2. Disable unused features using :ADConfig
3. Clear plugin cache: :Lazy clean
4. Update to latest Neovim version

KEYMAPS NOT WORKING					*adev-keymaps-not-working*

Problem: Custom keymaps don't respond.
Solution:
1. Check leader key setting in defaults.lua
2. Verify plugin is loaded: :Lazy
3. Check for key conflicts: :verbose map <leader>xx

PERFORMANCE ISSUES					*adev-performance*

Problem: Lag or freezing.
Solution:
1. Disable heavy plugins temporarily
2. Check memory usage
3. Update all plugins
4. Reduce LSP servers for current filetypes

FREQUENTLY ASKED QUESTIONS				*adev-faq*

Q: How do I add my own plugins?
A: Create files in lua/adev/custom-plugins/ following the Lazy spec format.

Q: Can I use a different color scheme?
A: Yes, set colorscheme using :ADConfig.

Q: How do I disable LSP for certain filetypes?
A: Use :ADConfig to set lsp.servers to exclude unwanted filetypes.

Q: Where are my snippets stored?
A: In the snippets/ directory. Edit or add .snippets files.

Q: How do I update Adev.nvim?
A: The update manager checks automatically. Use :AdevUpdate to force update.

MODULAR ARCHITECTURE					*adev-architecture*

Adev.nvim follows a modular design:
- lua/adev/: Core configuration
- lua/adev-common/: Shared utilities
- lua/adev-files/: Experimental features
- plugin/: Global settings and keymaps

ADDING PLUGINS						*adev-adding-plugins*

1. Create lua/adev/plugins/your-plugin.lua: >
return {
  {
    "author/plugin",
    opts = {},
    keys = {
      { "<leader>xx", "<cmd>PluginCommand<cr>", desc = "Description" }
    }
  }
}
<
2. Add to plugin documentation
3. Test lazy loading

CUSTOMIZING EXISTING PLUGINS			*adev-customizing-plugins*

Override plugin configs in lua/adev/config/: >

-- lspconfig.lua
return function()
  require("lspconfig").lua_ls.setup({
    settings = {
      Lua = {
        diagnostics = {
          globals = { "vim" }
        }
      }
    }
  })
end
<
